The Shelf UI MCP (Model Context Protocol) server exposes the component registry as structured, queryable tools. Once connected, your AI assistant can browse components, get install commands, and read prop documentation without leaving your editor.

## What the server exposes

| Tool | Description |
| --- | --- |
| `search_components` | Search the Shelf UI registry by name, category, or keyword |
| `get_component` | Get full documentation for a specific component — props, usage, examples |
| `get_install_command` | Generate the correct shadcn CLI command for a component and package manager |
| `list_adapters` | List available storage adapters and their config requirements |
| `get_adapter` | Get full documentation for a storage adapter |

## Planned connection

```json title="Desktop config (planned)"
{
  "mcpServers": {
    "shelf-ui": {
      "command": "npx",
      "args": ["-y", "@shelf-ui/mcp"]
    }
  }
}
```

No API key required. The server reads from the public Shelf UI registry.

## Example tool calls (planned)

When connected, you can ask your assistant:

> *"What file upload components does Shelf UI have?"*

The assistant will call `search_components({ query: "upload" })` and return a structured list with descriptions and install commands.

> *"How do I use the Dropzone with Supabase?"*

The assistant will call `get_component({ name: "dropzone" })` and `get_adapter({ name: "supabase" })` and compose a working integration example from the results.

## In the meantime

While the MCP server is in development, use the [Skills](/docs/skills) page to give your AI assistant Shelf UI context via a file-based approach that works today with Cursor, Claude Code, Windsurf, Antigravity and GitHub Copilot.
