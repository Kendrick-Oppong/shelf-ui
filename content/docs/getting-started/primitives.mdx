Every interactive Shelf UI component supports both **Radix UI** and **Base UI**.

When you install a component, Shelf UI automatically detects your project's existing primitive layer and installs the matching implementation. The component API remains identical regardless of which primitive library powers it.

## What are primitives?

Primitives are unstyled, accessible, headless components that handle behavior — keyboard navigation, focus management, ARIA attributes, screen reader announcements — without imposing visual opinions. Shelf UI components are built on top of them.

Shelf UI supports both major primitive ecosystems:

| | Radix UI | Base UI |
| --- | --- | --- |
| **Maintained by** | WorkOS | MUI team |
| **Used in** | shadcn/ui, Vercel, Linear | MUI ecosystem |
| **Maturity** | Production-proven | Rapidly growing |
| **Bundle size** | Tree-shakeable | Tree-shakeable |
| **React compatibility** | React 16+ | React 18+ |
| **WAI-ARIA** | Full compliance | Full compliance |
| **Animation** | Via CSS or Motion libraries | Via CSS or Motion libraries |

## How detection works

    ```bash title="terminal"

    ? Which implementation would you like to use?
    › Radix UI
      Base UI
    ```
  </Tab>
</Tabs>

## Consistent APIs

Regardless of which primitive implementation is installed, the public API remains identical:

```tsx title="Your component code"
import { Dropzone } from "@/components/shelf-ui/dropzone";

export function UploadArea() {
  return (
    <Dropzone
      onDrop={handleDrop}
      accept={["image/*"]}
      maxSize={10_000_000}
    />
  );
}
```

Only the internal implementation differs. Your application code never changes.

## Headless hooks

Every interactive component ships with its underlying hook. Use the hook directly if you want complete control over the UI.

```tsx title="Using the hook directly"
import { useFileUpload } from "@/hooks/use-file-upload";

export function CustomDropzone() {
  const {
    files,
    isDragging,
    getRootProps,
    getInputProps,
  } = useFileUpload({
    accept: ["image/*"],
    maxSize: 5_000_000,
    onDrop: (accepted) => {
      console.log(accepted);
    },
  });

  return (
    <div
      {...getRootProps()}
      data-dragging={isDragging}
      className="my-custom-zone"
    >
      <input {...getInputProps()} />

      {isDragging
        ? "Drop files here"
        : "Drag files or click to upload"}
    </div>
  );
}
```

The hook API is identical regardless of which primitive implementation is installed.

## FAQ

<Accordions type="single">
  <Accordion title="Do I need to choose between Radix UI and Base UI?">
    Usually not. Shelf UI detects your project's existing primitive layer and installs the matching implementation automatically.
  </Accordion>

  <Accordion title="How does detection work?">
    Shelf UI inspects your existing component foundation and dependencies to determine whether your project is built on Radix UI or Base UI.
  </Accordion>

  <Accordion title="Can I override the detected primitive layer?">
    Yes. If needed, you can explicitly choose a different implementation during installation.
  </Accordion>

  <Accordion title="Which primitive is more performant?">
    In practice, the difference is negligible. Choose based on ecosystem fit and consistency with your existing stack rather than performance.
  </Accordion>

  <Accordion title="What happens if a component doesn't need a primitive?">
    Components like FileCard, FileIcon, and StorageQuota are purely presentational and don't depend on either primitive library.
  </Accordion>
</Accordions>
