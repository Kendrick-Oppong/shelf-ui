Shelf UI is an open-source component library for building file-related interfaces in React. It gives you composable, copy-paste components designed for the complete file experience — upload, preview, manage, and navigate — built on React 19, Tailwind CSS v4, and Radix UI.

Think of it as **shadcn/ui, purpose-built for file UIs**.

## Why Shelf UI?

<Cards>
  <Card title="File-domain first">
    No other library treats file management as a complete UI domain. Shelf UI owns the entire experience — from the dropzone to the file tree.
  </Card>
  <Card title="Copy-paste ownership">
    Components are not installed as a dependency. You copy them into your project and own them entirely. Edit, extend, delete — no abstraction in the way.
  </Card>

  <Card title="Storage adapters built in">
    First-class Supabase Storage, AWS S3, and Cloudinary support. Drop in your config and uploads just work.
  </Card>
  <Card title="Headless hooks included">
    Every component ships with its underlying hook. Use the UI layer or go fully headless — your choice.
  </Card>
  <Card title="Shadcn registry native">
    Built on the [shadcn open registry spec](https://ui.shadcn.com/docs/registry). Install any component with a single CLI command. Peer dependencies resolved automatically.
  </Card>
</Cards>

## How it works

Shelf UI is built on the [shadcn registry](https://ui.shadcn.com/docs/registry). Components are not installed as a package — you copy them into your project and own them entirely.

<Tabs groupId="install" items={["npm", "pnpm", "yarn", "bun"]} persist>
  <Tab value="npm">
```bash title="terminal"
npx shadcn@latest add @shelf-ui/dropzone
```
  </Tab>
  <Tab value="pnpm">
```bash title="terminal"
pnpm dlx shadcn@latest add @shelf-ui/dropzone
```
  </Tab>
  <Tab value="yarn">
```bash title="terminal"
yarn dlx shadcn@latest add @shelf-ui/dropzone
```
  </Tab>
  <Tab value="bun">
```bash title="terminal"
bunx shadcn@latest add @shelf-ui/dropzone
```
  </Tab>
</Tabs>

This copies the component source into your `components/shelf-ui` directory. Customize it however you want.

More components are being actively developed. Follow the repo for updates.

## FAQ

<Accordions type="single">
  <Accordion title="Do I need to install anything globally?">
    No global install required. Each component installs its own peer
    dependencies (Radix UI, Tailwind, Lucide) via the shadcn CLI.
    Run one command per component.
  </Accordion>

  <Accordion title="Does this work with my existing shadcn setup?">
    Yes. Shelf UI is a separate registry that integrates with your existing
    `components.json`. Add the registry alias once and all Shelf UI
    components install with the same CLI you already use.
  </Accordion>
  <Accordion title="Can I use only some components?">
    Yes — that's the point. Install only what you need. Components are
    independent
  </Accordion>
  <Accordion title="Can I use Shelf UI without Next.js?">
    Yes. Shelf UI components work in any React 19+ project — Vite, Remix,
    TanStack Start. The shadcn CLI handles the file copying regardless of
    your framework.
  </Accordion>
  <Accordion title="Are the components accessible?">
    Yes. Components built on Radix UI primitives are WAI-ARIA
    compliant out of the box. All interactive components support full
    keyboard navigation.
  </Accordion>
</Accordions>
